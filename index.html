What will the next couple labs be like?

The next labs will focus on "client side technologies" which are sometimes called "front-end technologies". They include things like different popular libraries and other frameworks for web browsers.

Most browser technologies boil down to the three main technologies:

  • HTML
  • CSS
  • Scripting
    1. JavaScript
    2. CoffeeScript
    3. black magic

What will we learn in this lab?

In this lab we will get a head start on laying down the basics by learning how to use HTML, as well as your browser's DevTools (developer tools).

HTML helps you structure web pages both visualy and with proper accessibility.

Accessibility refers to how and what can access and understand your webpage. For example, a properly structured HTML makes it easier for things like search engines and web crawlers to find and understand your webpages.

Who decides what HTML does?

HTML specifications is a document that compiles and describes how and what different HTML tags do. HTML tags are the name for some HTML field that denots some specific functionality. The browser companies then look at the specifications and develop their web browsers to display those tags according to the established specification.

DevTools?

Dev tools are different, but are usually pretty standard across different browsers. Because browser technologies often do not crash when they encounter an unexpected syntax, and instead try thier best to work around it, it can often make developemnt hard and unpredictable and it is thus very important to get to know how these debugging tools work and function.

Wrap Up

  • Keep ahead, don't keep up by deeply learning HTML/CSS/JS
  • Write your HTML with structure and accessibility in mind
  • HTML syntax is a standard that browsers follow and so sould you
  • Proper HTML will bring out the best in your browser
